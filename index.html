    Програмні засоби та інструменти

    1. Java SE Development Kit 8 - комплект розробника застосунків на мові Java (див. файл se_appendix_1.pdf).

    2. Eclipse IDE for Java Developers - середовище розробки. Додатково встановити:

      • ObjectAid UML Explorer update site URL. Встановити тільки ObjectAid Class Diagram plug-in - плагін візуалізації кода (показує UML class diagram);

      • Checkstyle update site URL - перевіряє початковий код на відповідність стандартам кодування (Tutorial);

      • Subclipse update site URL 1 - Eclipse plug-in which provides support for Subversion (Project home). Встановити компоненти:

        • Subclipse;

        • Subversion Client Adapter;

        • Subversion JavaHL Native Library Adapter;

      • Regex Util update site URL - для зручної перевірки регулярних виразів (Project home). Відкрити в Eclipse за допомогою меню Window / Show View / Other / Regex Util.

    3. TortoiseSVN 1 - Subversion (SVN) client. В лабораторіях ОЦ НТУ "ХПІ" включити використання proxy-сервера в меню TortoiseSVN/Settings/Network:

      • Enable Proxy Server;

      • Server address: 172.17.10.2;

      • Port: 3128.

    4. SVN Hosting 1 - веб-сервіс управління версіями:

      • потрібна попередня реєстрація GET STARTED WITH FREE REPOSITORIES;

      • при реєстрації обов'язково вказувати Username у вигляді surname_name - прізвище та ім'я розробника латинкою в нижньому регістрі;

    Вимоги

    1. Рішення всіх лабораторних робіт розміщувати у відповідних пакетах єдиного проекту Eclipse:

      • проект повинен мати назву surname - прізвище розробника латинкою в нижньому регістрі;

      • пакети, що містять всі розроблені класи та підпакети для кожного рішення, повинні мати назву ua.khpi.oop.surnameXX, де XX - номер роботи.

    2. Початковий код повинен відповідати Java Code Conventions.

    3. Коментарі використовувати для автоматичної генерації документації за допомогою Javadoc, вікі.

    4. Схему класів (class diagram) обов'язково розміщувати в кореневому пакеті кожного рішення (png-рисунки).

    Робота з репозиторієм 1

    1. Початковий код (лише той, що успішно компілюється) розташовувати в директоріях surname/src у відповідних пакетах, де surname - назва особистої директорії в спільному сховищі, що відповідає назві проекту Eclipse.

    2. Звіти розташовувати в директоріях surname/doc/surnameXX, де XX - номер рішення.

    3. Повідомлення до коміту - що саме ви зберігаєте - має бути зрозумілим. У заголовку повідомлення вказувати:

      • номер тікета (завдання) - номер у таблиці, що містить список спільних завдань та зауважень;

      • тему/компонент до якого належить коміт. Наприклад:

        #12 shevchenko01: завдання виконано, звіт готовий до перевірки

        #17 Locator: алгоритм пошуку оптимізовано

        #22, #23 Task01: відформатовано текст, додано коментарів

    4. Командою Team / Share Project with SVN Repository можна помістити в репозиторій проект Eclipse:

      • в діалоговому вікні обрати Use existing repository location;

      • вказати Checkout URL спільного сховища.

    5. Checkout URL для доступу до особистої директорії формується додаванням до Checkout URL спільного сховища назви особистої директорії. Наприклад, загальне сховище має Checkout URL:

      https://subversion.assembla.com/svn/kitXX-2015-oop/

      Тоді Checkout URL для доступу до особистої директорії shevchenko:

      https://subversion.assembla.com/svn/kitXX-2015-oop/shevchenko

    6. Приклад структури репозиторія проекта з назвою xone, якщо Checkout URL спільного сховища

    ВИСНОВКИ Заключення стосовно повноти досягнення мети

    1. Типи даних, літерали, операції і оператори

    Мета

    Ознайомлення з платформою Java SE та середовищем розробки Eclipse IDE.

    Загальне завдання

    Вирішити три прикладні задачі на мові Java в середовищі Eclipse.

    Прикладні задачі

    1. Знайти найбільший спільний дільник двох цілих позитивних чисел.

    2. Знайти суму цифр заданого цілого числа.

    3. Перевірити, чи є задане число простим (тобто не ділиться без залишку на жодні числа, крім себе і 1).

    2. Обробка масивів і рядків. Інтерактивні консольні програми

    Мета

    • Набуття навичок вирішення прикладних задач з використанням масивів і рядків.

    • Реалізація діалогового режиму роботи з користувачем в консольних програмах.

    Загальне завдання

    1. Розробити консольну програму для вирішення прикладної задачі (Class String, Manipulating Characters in a String, Comparing Strings and Portions of Strings).

    2. Забезпечити діалоговий режим роботи з користувачем (введення, результат, вихід і т.д.).

    3. Використовувати параметри командного рядка для визначення режиму роботи програми.

    4. При вирішенні прикладних задач використовувати латинку.

    5. Для обробки даних використовувати класи-утиліти (Utility Class), продемонструвати використання об'єктів класу StringBuilder або StringBuffer.

    Зауваження

    • У програмі має бути реалізовано текстове меню, яке містить команди для визначення користувачем того чи іншого режиму роботи: введення даних; перегляд даних; виконання обчислень; перегляд результату.

    • Крім меню, для визначення режиму роботи та інших вхідних даних, користувач може використовувати параметри запуску консольної програми в командному рядку.

    Прикладні задачі

    1. Ввести несколько строк. Разбить строки на три группы: начинается с гласной; начинается с согласной; начинается не с буквы. Найти самую короткую строку в каждой группе. Вывести эту строку и ее длину.

    2. Ввести несколько строк. Упорядочить и вывести строки по алфавиту (первый приоритет) и в порядке возрастания их длин (второй приоритет).

    3. Ввести несколько строк. Разбить на две группы: строки, длина которых меньше средней; строки, длина которых не меньше средней. Вывести строки и их длины.

    4. Ввести текст. В каждом слове текста k-ю букву заменить заданным символом. Если k больше длины слова, выдать сообщение и корректировку не выполнять. Исходные данные и результат вывести в виде таблицы.

    5. Ввести текст. В тексте каждую букву заменить ее номером в алфавите. Вывести результат следующим образом: в одной строке печатать текст с двумя пробелами между буквами, в следующей строке внизу под каждой буквой печатать ее номер.

    6. Ввести текст. Из текста удалить все символы, кроме пробелов, не являющиеся буквами. Повторяющиеся пробелы заменить на одиночные. Между последовательностями букв разделённых непробелами (знаками препинания), оставить хотя бы один пробел. Вывести исходный текст и результат.

    7. Ввести текст. Из текста удалить все слова заданной длины, начинающиеся на согласную букву. Вывести исходный текст и результат.

    8. Ввести текст. В тексте найти все пары слов, из которых одно является обращением (словом наоборот) другого. Результат вывести в виде таблицы.

    9. Ввести текст. Найти и напечатать, сколько раз повторяется в тексте каждое слово. Результат вывести в виде таблицы.

    10. Ввести текст. Найти, каких букв, гласных или согласных, больше в каждом предложении текста. Результат вывести в виде таблицы.

    11. Ввести текст. В тексте слова заданной длины заменить указанной подстрокой. Вывести исходный текст и результат.

    12. Ввести текст. Текст разбить на предложения. Для каждого предложения вывести слова, у которых первая и последняя буквы совпадают. Результат вывести в виде таблицы.

    13. Ввести текст. Текст разбить на предложения. Для каждого предложения найти и напечатать все слова максимальной и все слова минимальной длины. Результат вывести в виде таблицы.

    14. Ввести текст. После каждого слова текста, заканчивающегося заданным символом, вставить указанное слово. Вывести исходный текст и результат.

    15. Ввести текст. В тексте найти и вывести все слова-палиндромы.

    3. Розробка власних контейнерів. Серіалізація/десеріалізація об'єктів

    Мета

    • Набуття навичок розробки власних контейнерів.

    • Ознайомлення з принципами серіалізації/десеріалізації об'єктів.

    Загальне завдання

    1. Розробити клас-контейнер, що ітерується та серіалізується для збереження початкових даних завдання л.р. №2 у вигляді масиву об'єктів з можливістю додавання, видалення і зміни елементів.

    2. Реалізувати і продемонструвати тривале зберігання/відновлення контейнера за допомогою серіалізації/десеріалізації (Tutorial).

    3. Обмінятися відкомпільованим (без початкового коду) службовим класом (Utility Class) рішення задачі л.р. №2 з іншим студентом (визначає викладач).

    4. Продемонструвати вирішення двох варіантів (власного і отриманого за обміном) прикладних задач л.р. №2 для послідовної та вибіркової обробки елементів розробленого контейнера.

    5. Розробити консольну програму та забезпечити діалоговий режим роботи з користувачем для демонстрації та тестування рішення.

    4. Інкапсуляція. Основи введення/виведення

    Мета

    • Використання об'єктно-орієнтованого підходу для вирішення прикладної задачі.

    • Освоєння основ управління введенням/виведенням даних.

    Загальне завдання

    1. Використовуючи принципи інкапсуляції, реалізувати клас для представлення сутності відповідно прикладної задачі.

    2. Забезпечити і продемонструвати коректне введення та відображення кирилиці.

    3. Продемонструвати можливість управління масивом об'єктів розробленого класу.

    4. Забезпечити можливість збереження і відновлення масива об'єктів, не використовуючи протокол серіалізації.

    5. Забезпечити діалог з користувачем у вигляді простого текстового меню.

    6. При збереженні та відновленні даних забезпечити діалоговий режим вибору директорії з відображенням вмісту і можливістю переміщення по підкаталогах.

    Прикладні задачі

    1. Кадровое агентство. Данные о соискателе: регистрационный номер; опыт работы - набор значений "cпециальность, стаж"; образование; дата увольнения; требования к будущей работе - набор необязательных свойств вида "cпециальность, условия труда, минимальная зарплата".

    2. Кадровое агентство. Данные о вакансии: фирма; cпециальность; условия труда; оплата; требования к специалисту - набор необязательных свойств вида "cпециальность, стаж, образование".

    3. Адресная книга. Запись: Ф.И.О.; дата рождения; телефоны (количество не ограничено); адрес; дата и время редактирования.

    4. Библиотека. Данные о книге: ISBN; название; авторы (количество не ограничено); издательство; жанр; дата издания.

    5. Справочник покупателя. Торговая точка: название; адрес; телефоны (количество не ограничено); специализация; время работы (с указанием дней недели).

    6. Полицейская картотека. Данные о преступнике: Ф.И.О.; дата рождения; даты судимостей (количество не ограничено); дата последнего лишения свободы; дата последнего освобождения.

    7. Бюро знакомств. Запись о клиенте: пол; регистрационный номер; дата регистрации; сведения о себе (произвольный набор свойств: имя, рост, цвет глаз, дата рождения, хобби и т.д.); требования к партнеру (произвольный набор свойств).

    8. Автостанция. Запись в расписании: номер рейса; время отправления; дни недели; количество свободных мест; маршрут - неограниченный набор значений вида "название станции, время прибытия".

    9. Билетная касса. Данные о маршруте: маршрут - неограниченный набор значений вида "название станции, время прибытия (для промежуточных и конечной), время отправления (для начальной и промежуточных), количество свободных мест"; общее количество мест; дни недели; номер рейса.

    10. Отдел кадров. Личное дело сотрудника: паспортные данные; образование; оклад; карьера - набор значений "дата назначения, должность, отдел"; характеристика - набор свойств и оценок.

    11. Магазин. Запись в каталоге товаров: наименование; единица измерения; количество; цена единицы; дата поступления; описание (неограниченный набор характеристик вида "свойство, значение").

    12. Аудиотека. Композиция: название; жанр; исполнитель; текст (если есть); дата создания; длительность; формат данных; рейтинг (набор характеристик вида "свойство, значение").

    13. Гостиница. Заказ номера: паспортные данные; даты поселения и выселения; номер (класс, число мест); причина поселения (количество не ограничено).

    14. Планировщик. Мероприятие: дата, время и протяженность; место проведения; описание; участники (количество не ограничено).

    15. Автосалон. Автомобиль: марка, год выпуска, технические характеристики в виде "параметр - значение" (среди которых расход топлива в городском и загородном цикле), техническое состояние, запрашиваемая цена.

    5. Параметризація в Java. Розробка параметризованих контейнерів

    Мета

    1. Вивчення принципів параметризації в Java.

    2. Розробка параметризованих класів та методів.

    Загальне завдання

    Використовуючи програму рішення завдання лабораторної роботи №4:

    1. Створити власний клас-контейнер, що параметризується (Generic Type) для реалізації колекції об'єктів на основі зв'язних списків.

    2. Для розроблених класів-контейнерів забезпечити можливість використання їх об'єктів у циклі foreach в якості джерела даних.

    3. Створити клас утиліт, що надає параметризовані методи (Generic Methods) для обробки контейнерів об'єктів згідно прикладної задачі.

    4. Забезпечити можливість збереження та відновлення контейнера об'єктів, не використовуючи протокол серіалізації.

    5. Продемонструвати розроблену функціональність (створення, управління та обробку власних контейнерів) в діалоговому та автоматичному режимах.

    Зауваження

    • Заборонено використання контейнерів (колекцій) і алгоритмів з Java Collections Framework.

    • Автоматичний режим виконання програми задається параметром командного рядка -auto. Наприклад, java ClassName -auto.

    • В автоматичному режимі діалог з користувачем відсутній, необхідні данні генеруються, або зчитуються з файлу.
